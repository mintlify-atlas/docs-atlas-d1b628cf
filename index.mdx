# Class Variance Authority

**CVA** is a tiny (~1.6KB) TypeScript utility that brings first-class variant support to any CSS approach — Tailwind CSS, CSS Modules, plain CSS, or BEM.

CSS-in-TS libraries like [Stitches](https://stitches.dev/docs/variants) and [Vanilla Extract](https://vanilla-extract.style/documentation/api/style-variants/) are fantastic for building type-safe UI components, but they require you to fully adopt their styling system. CVA gives you the same developer experience — type-safe variants, compound variants, and default values — while letting you write CSS however you prefer.

```typescript
import { cva } from "cva";

const button = cva({
  base: "font-semibold border rounded",
  variants: {
    intent: {
      primary: "bg-blue-500 text-white border-transparent hover:bg-blue-600",
      secondary: "bg-white text-gray-800 border-gray-400 hover:bg-gray-100",
    },
    size: {
      small: "text-sm py-1 px-2",
      medium: "text-base py-2 px-4",
    },
  },
  defaultVariants: {
    intent: "primary",
    size: "medium",
  },
});

button({ intent: "secondary", size: "small" });
// => "font-semibold border rounded bg-white text-gray-800 border-gray-400 hover:bg-gray-100 text-sm py-1 px-2"
```

## Why CVA?

Creating variants with traditional CSS approaches is tedious — manually matching classes to props, duplicating logic, and writing TypeScript types by hand. CVA automates all of that.

## Packages

CVA is available as two packages:

| Package | Description |
|---------|-------------|
| [`cva`](https://www.npmjs.com/package/cva) | Current package — beta channel with latest features |
| [`class-variance-authority`](https://www.npmjs.com/package/class-variance-authority) | Legacy package — stable v0.x releases |

<Note>
  New projects should use the `cva` package. The `class-variance-authority` package is maintained for backward compatibility.
</Note>
